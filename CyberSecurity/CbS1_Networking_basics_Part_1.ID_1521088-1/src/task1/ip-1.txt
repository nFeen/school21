2) Представь, что тебе нужно выделить подсети под различные сегменты организации N — технологический (3 машины), серверный (16 машин) и пользовательский (из 32 машин).

Для каждой подсети нужно количество адресов равное количеству машин плюс два. Так как помимо адресов самих машин есть еще broadcast адрес и адрес сети.
Подсеть технологического сегмента: 5 адресов.
Подсеть серверного сегмента: 18 адресов.
Подсеть пользовательского сегмента: 34 адреса.

3) Сети с какими масками необходимо выделить для каждого из сегментов? Приведи обоснование своего ответа.

Нам нужно определить маску, которая предоставляет минимально допустимое количество адресов для нашей подсети. Это нужно, чтобы не использовать лишние адреса. Чтобы определить маску нам нужно найти степень двойки, при возведении в которую получится минимальное число близкое к количеству нужных нам адресов.
Для технологического отдела (3 машины): необходимо 5 адресов, маска будет /29 (255.255.255.248). Данная маска подсети дает нам 8 адресов.
Для серверного отдела (16 машин): необходимо 18 адресов, маска будет /27 (255.255.255.224). Данная маска подсети дает нам 32 адреса.
Для пользовательского отдела (32 машины): необходимо 34 адреса, маска будет /26 (255.255.255.192). Данная маска подсети дает нам 64 адреса.
